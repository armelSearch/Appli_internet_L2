TP1 – Validation des champs d'un formulaire en JavaScript

Introduction

L'objectif de ce TP est vérifier la conformité des champs d'un formulaire en Javascript dans le navigateur avant de l'envoyer.

Dans ce TP vous verrez comment :

  • détecter une action sur un champ d'un formulaire ;
  • utiliser les expressions rationnelles ;
  • agir dynamiquement sur le DOM et le CSS.

Environnement de développement

Vous aurez besoin :

  • d'un éditeur de code pour HTML, CSS et Javascript : gedit, sublime-text, atom.
  • de la console Javascript du navigateur pour afficher :
    • les messages d'erreur
    • et la sortie de la fonction console.log()
    raccourcis pour accéder à la console :
    • Sous Firefox : Ctrl-Maj-K
    • Sous Chrome : Ctrl-Maj-J
  • Notez qu'il est possible d'éditer le code Javascript directement dans Chrome : Ctrl-Maj-I

Exercice

Pour ce TP, nous vous fournissons un formulaire dont la structure et la présentation sont déjà faites. Vous n'aurez pas besoin de les modifier. Par contre, il vous faudra compléter le code JavaScript.

Pour commencer, téléchargez les fichiers form.html , form.css et form.js dans un dossier L2/DAI/TP1/.

Visualisez le formulaire dans votre navigateur Web et assurez-vous que la feuille de style est bien prise en compte.

Comportement du formulaire

Un clic sur le bouton "s'inscrire" déclenche les tests de validation des champs du formulaire.

Quand le contenu d'un champ n'est pas correct, un message d'erreur s'affiche sous le champ.

Le script fourni affiche tous les messages d'erreur en cas de clic sur le bouton. Mais seul la vérification du champ "Identifiant" a été implémentée.

Quand un champ a été saisi, le fait de quitter ce champ déclenche la vérification de sa validité (événement "blur").

Vérifier la validité des champs

Pour cela, vous écrirez des couples de fonction valider...() et vérifer...(). Un premier exemple est donné dans le code pour la vérification et la validation de l'identifiant qui doit avoir au moins 12 caractères constitués juste de lettres.

Pour faire cette vérification, nous avons utilisé la fonction test() qui permet de tester si une chaîne de caractères vérifie une expressions rationnelles.

Vous écrirez les fonctions qui permettent de vérifier si :

  1. L'âge saisi est supérieur ou égal à 18.
    • Attention : "vingt ans" n'est pas correct !
  2. Le mot de passe doit être constitué d'au moins :
    • 8 caractères
    • une lettre minuscule (expression rationnelle : /[a-z]/)
    • une lettre majuscule
    • un chiffre
    • un caractère qui n'est pas une lettre ni un chiffre
  3. La longueur de la barre de progression dépend de la force du mot de passe. (exemple : barre de progression).
    Chacun des critères ci-dessus contribue à 20% dans la force du mot de passe.
    Les couleurs de la barre dépendent de la force du mot de passe :
    • En dessous de 40%, la barre est rouge
    • De 40% à 70%, la barre est orange
    • En dessus de 70%, la barre est verte
  4. Testez si les deux mots de passe saisis par l'utilisateur sont identiques.
  5. Vérifiez si l'utilisateur a accepté les conditions générales d'utilisation (CGU).

Conseils pour la programmation

  • Toujours vérifier ce qui s'affiche dans la console. Si une erreur est reportée, la corriger avant de poursuivre.
  • Programmez de manière incrémentale : écrivez quelques instructions et vérifiez qu'elles produisent bien l'effet escompté.
  • Assurez-vous que les variables que vous manipulez contiennent bien la valeur que vous pensez. Affichez-les avec console.log() pour en être sûr.
