TP1 – Création et validation d'un formulaire en JavaScript

Introduction

L'objectif de ce TP est de construire et de valider dynamiquement un formulaire en Javascript.

Dans ce TP vous verrez comment :

  • détecter une action sur un champ d'un formulaire ;
  • utiliser les expressions régulières ;
  • agir dynamiquement sur le DOM ou le CSS.

Environnement de développement

Vous aurez besoin :

  • d'un éditeur de code pour HTML, CSS et Javascript : gedit, sublime-text, atom.
  • de la console Javascript du navigateur pour afficher :
    • les messages d'erreur
    • et la sortie de la fonction console.log()
    raccourcis pour accéder à la console :
    • Sous Firefox : Ctrl-Maj-K
    • Sous Chrome : Ctrl-Maj-J
  • Notez qu'il est possible d'éditer le code Javascript directement dans Chrome : Ctrl-Maj-I

Bonnes pratiques

Voici quelques règles pour écrire proprement du code Javascript :

  • Externaliser le plus possible le code Javascript dans des fichiers .js liés au document HTML : 
    <script type="text/javascript" src="./code.js"></script>
  • Écrire le code Javascript de manière modulaire en le structurant en autant de fonctions que nécessaires pour les maintenir courtes (pas plus de 20-30 lignes idéalement) ;
  • Ces fonctions sont appelées dans un "script principal" dans la page HTML : 
    <script>
  var entier = 2;
  var resultat = carre(entier); // appel de ma fonction "carre";
  var message = "le carre de " + entier + " est " + resultat;
  console.log(message); // affichage dans la console
</script>

Pour commencer ...

Dans ce TP, vous complèterez un formulaire dont la structure et la présentation sont déjà faites : form.html et form.css .


  1. Copiez ces deux fichiers dans un dossier L2/DAI/TP1/ ;
  2. Visualisez le formulaire dans votre navigateur Web et assurez-vous que la feuille de style est bien prise en compte.

Entrainements

A - Réagir aux événements sur le bouton

  1. Affichez un message à la console lorsque l'utilisateur clique sur le bouton de soumission du formulaire.
    Vous pourrez vous inspirer de ceci.
  2. Lancez une alerte qui affiche le message précédent lorsque le bouton est cliqué.
    Vous pourrez vous inspirer de ceci.
  3. Recupérez le contenu du bouton et affichez le à la console.
    Vous pourrez vous inspirer de ceci.
  4. Grisez le bouton de soumission du formulaire


B - Vérifier la validité des champs

Ecrivez des fonctions qui permettent de vérifier si :

  1. L'âge saisi est supérieur ou égal à 18.
    NB : La fonction onblur permet de détecter des changements dans un élément du DOM.
  2. L'identifiant a au moins 12 caractères constitués juste de lettres.
    NB : La fonction test permet de tester si une chaîne de caractères vérifie une expression régulière ;
  3. Ajouter dans le DOM un élément HTML de type barre de progression. Son objectif est de jauger la force du mot de passe. Chacun des critères ci-dessous contribue à 20% dans la force du mot de passe, qui doit contenir :
    • au moins 8 caractères
    • au moins une lettre minuscule
    • au moins une lettre majuscule
    • au moins un chiffre
    • au moins un caractère qui n'est pas une lettre ni un chiffre

    Les couleurs de la barre dépendent de la force du mot de passe :
    • En dessous de 40%, la barre est rouge
    • De 40% à 70%, la barre est orange
    • En dessus de 70%, la barre est verte
  4. Rajouter une fonction qui permet de tester si les deux mots de passe saisis par l'utilisateur sont identiques.
  5. Écrivez une fonction qui vérifie si l'utilisateur a accepté les conditions générales d'utilisation (CGU).


C- Guider l'utilisateur

Dans la cas où l'utilisateur n'a pas rempli correctement le formulaire :

  • ajouter en dessous du champ un texte qui spécifie l'erreur ;
  • ce texte doit apparaitre en rouge avec une petite police ;
  • une fois que l'erreur est corrigée, le texte disparaît ;
  • si aucune erreur détectée, le bouton se dégrise.
NB : pour faire disparaître un élément d'id myID du DOM d'une page, il suffit de faire : 
<script>
   document.getElementById('myID').style.display = 'none'
</script>
Pour le faire apparaître, remplacer 'none' par 'inline'
